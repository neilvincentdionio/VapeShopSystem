E-Commerce Vape Shop System
Laboratory Exercise 3 – System Development: Login, Dashboard, and Records Module

Project Overview
The E-Commerce Vape Shop System is a web-based application developed as part of Laboratory Exercise 3. The system implements three core modules:
•	Login Module
•	Dashboard Module
•	Records Module
The system supports role-based access for:
•	Admin
•	Customer/User
This project demonstrates secure authentication, dashboard interface design, database record management, and version control collaboration.

Objectives
•	Implement secure user authentication
•	Develop role-based dashboard access
•	Create a functional records module with CRUD operations
•	Integrate all modules into one system
•	Document development using GitHub

System Modules
1. Login Module
Features:
•	Session-based authentication
•	Credential validation using database
•	Login error handling
•	Role identification (Admin, Customer)
•	Secure password handling
Process:
1.	User enters email and password
2.	System validates credentials
3.	If valid → Session created
4.	Redirect to role-based dashboard

2. Dashboard Module
The dashboard provides a user-friendly interface depending on the user role.
Admin Dashboard
•	Manage users
•	View sales reports
•	Monitor system activity
Customer Dashboard
•	Browse vape products
•	Search and filter items
•	Add to cart
•	Checkout and place orders

3. Records Module
This module handles database operations.
Functions Implemented:
•	Create records
•	Read records
•	Update records
•	Delete records
•	Search and filter functionality
Database Stores:
•	Users
•	Records
